# Plan 02-01 — Baseline profiling & hot-path map

## Objective

Зафиксировать **воспроизводимый baseline** производительности на представительном большом дереве (кэш или повторный скан): где тратится время в CPU (фильтры, layout, подготовка к рендеру) и где в GPU (wgpu submit, pipeline switches). Результат — **markdown-отчёт** в `.planning/phases/02-performance-gpu-at-scale/` с 1–3 главными узкими местами и предложенными следующими шагами.

## must_haves

- Файл `PERF-BASELINE.md` существует и содержит команды измерения, окружение (OS, GPU), и именованные hotspots.
- Ни одного обязательного изменения поведения приложения в этом плане (только измерения и, при необходимости, **временные** `tracing::span` без изменения логики).

## Verification

- `PERF-BASELINE.md` содержит строку `## Hotspots` и минумум три подпункта или таблицу с колонкой «Компонент».
- `grep -q "## Hotspots" .planning/phases/02-performance-gpu-at-scale/PERF-BASELINE.md` завершается с кодом 0.

<threat_model>
Локальные профилировщики не открывают сетевых портов. Риск: случайный commit отладочного спама — не коммитить `println!` в горячий путь; использовать `tracing` с feature gate или удалить перед PR.
</threat_model>
